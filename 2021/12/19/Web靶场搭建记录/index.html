auto和allow-hotplug的区别

参考

注意到接口有auto和allow-hotplug两种区别

auto：在系统启动的时候启动网络接口，无论网络接口有无连接 (插入网线)。如果该接口配置了 DHCP，则无论有无网线，系统都会去获取 DHCP。并且如果没有插入网线，则等该接口超时后才会继续 DHCP。配置这个命令，仅仅是用于开机启动时启动网络接口，如果不配置重启自动不会启动网络接口，就直接导致远程登录失败。

allow-hotplug：只有当内核从网络接口检测到热插拔事件后才会启动该接口。如果系统开机时该接口没有插入网线，则系统不会启动该接口。系统启动后，如果插入网线，系统会自动启动该接口。配置这个命令，是为了保证端口状态及时更新，或者避免由于手动操作导致的重启 network 失败。

软件安装

安装软件之前，先执行一遍apt-get update更新下源

SSH客户端自带：openssh-client（可以重新安装来更新）
SSH服务端：openssh-server

安装之后的配置文件在/etc/ssh

Apache2：apache2
具体是2.4的版本

MySQL

MySQL安装比较麻烦
Debian默认的apt库中没有它的包，需要额外安装添加
首先下载存储库（具体版本看MySQL官网）：
wget https://dev.mysql.com/get/mysql-apt-config_0.8.20-1_all.deb
然后安装：dpkg -i mysql-apt-config_0.8.20-1_all.deb
之后会进入一个安装页面，选择MySQL相关apt库及版本
安装完后记得apt-get update更新下源
然后apt-get install mysql-server安装服务端
（客户端是mysql-client，似乎安装服务端时已默认安装）
安装完毕之后应该会自启动了，查看MySQL服务状态：
systemctl status mysql

PHP

安装PHP7.4

PHP：php
这样子安装只能默认安装PHP7.4
另外会自动安装和apache2连接的包libapache2-mod-php7.4等
MySQL模块需要另外安装：
apt-get install php-mysql
会自动安装对应版本的连接模块

之后重启apache2服务

写个PHP探针，攻击机访问，OK！

然后还有curl、git啥的。。。。

安装PHP5.6

有些靶场环境只适配老版本的PHP，因此需要安装一个PHP5.6
（没理由叫我把靶场代码全改了吧。。。。）

安装依赖：
apt-transport-https、lsb-release、ca-certificates
有些可能已经安装了

安装apt源的证书：
sudo wget -O /etc/apt/trusted.gpg.d/php.gpg https://packages.sury.org/php/apt.gpg

添加apt源：
echo "deb https://packages.sury.org/php/ $(lsb_release -sc) main" > /etc/apt/sources.list.d/php.list
注意$(lsb_release -sc)不能通过直接编辑php.list来加
因为系统会执行lsb_release -sc命令，再把结果写到php.list

更新源：apt-get update

安装：apt-get install php5.6

还有其他的一些模块啥的自己看着安装
php5.6-mysql、php5.6-gd等

redis

直接apt-get install redis-server

善后及优化

后面建立网站的时候，注意把网站的权限和属组都变成apache2的默认用户
www-data

具体：
改变属主：chown -R www-data www
改变属组：chgrp -R www-data www

/usr/lib/systemd/system

/lib/systemd/system -> /usr/lib/systemd/system

![image-20230422153253336](md.image/[NOTE] Web靶场搭建记录/image-20230422153253336.png)

读取顺序：/etc/systemd/system、/usr/lib/systemd/system

systemctl enable命令用于在上面两个目录之间建立符号链接，相当于开机启动

下列目录优先级依次从高到低

Path Description
/etc/systemd/system Local configuration
/run/systemd/system Runtime units
/lib/systemd/system Units of installed packages

service设定文件：

  • /usr/lib/systemd/system/*.service: 预设设定文件
  • /etc/systemd/system/system/*.service.d/: 用户个性化设定文件，会被加入设定
  • /etc/systemd/system/system/*.service.wants/: 连结文件，启动该服务后推荐启动的其他服务
  • /etc/systemd/system/system/*.service.requires/: 连结文件，启动该服务之前，需要预先启动的服务
  • /run/systemd/generator.late: 使用systemd-sysv-generator工具处理SysV init脚本（即：/etc/init.d/*），自动生成.service文件

.service文件

Link：linux systemd service 服务文件简介

实际上使用systemctl命令操纵的是.service文件

查看系统上所有的服务列表：systemctl list-unit-files

杀死一个服务的所有子进程：systemctl kill <service>

关于systemd的其他知识可以看看鸟哥私房菜

Redis配置

路径：/etc/redis/redis.conf

允许远程主机访问数据库：
NETWORK一栏，注释掉bind 127.0.0.1 ::1
同时设置protected-mode no

说明是这样的：

![image-20230422111835635](md.image/[NOTE] Web靶场搭建记录/image-20230422111835635.png)

意思是Redis仅允许有bing的IP地址的远程连接，如果没有任何bind的地址，则允许互联网上任意IP链接数据库，而这样子很危险

Redis默认端口是6379

设置密码：SECURITY一栏，设置requirepass <yourpassword>

![image-20230422113202928](md.image/[NOTE] Web靶场搭建记录/image-20230422113202928.png)

有一个很有意思的说明：

![image-20230422113253225](md.image/[NOTE] Web靶场搭建记录/image-20230422113253225.png)

意思是Redis非常非常快，这使得暴力破解的效率可以高得离谱，因此建议使用诸如/dev/urandom这样的长随机字符串作为密钥
